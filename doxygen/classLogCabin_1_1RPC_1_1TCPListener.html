Detailed Description

A TCPListener listens for incoming TCP connections and accepts them.

The client should inherit from this and implement the handleNewConnection() method for when a connection is accepted.

TCPListeners can be created from any thread, but they will always run on the thread running the Event::Loop.

TODO(ongaro): Merge this class into RPC::OpaqueServer. It was originally separate to hide some libevent internals, but that's no longer the case.

std::string LogCabin::RPC::TCPListener::bind ( const RPC::Address &  listenAddress)

Listen on a new address.

You can call this multiple times to listen on multiple addresses. (But if you call this twice with the same address, the second time will always throw an error.) This method is thread-safe.

Parameters:
