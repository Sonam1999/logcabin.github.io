00035 /**
00036  * A MessageSocket is a message-oriented layer on top of a TCP connection.
00037  * It sends and receives discrete chunks of data identified by opaque IDs.
00038  * Higher layers can use this to build an RPC framework, both on the client
00039  * side and on the server side.
00040  *
00041  * On the wire, this adds a 12-byte header on all messages. The first 8 bytes,
00042  * encoded as an integer in big endian, are the message ID. The next 4 bytes,
00043  * encoded as an integer in big endian, are the number of bytes of data
00044  * following the header. Following the header, the data is sent as an opaque
00045  * binary string.
00046  *
00047  * TODO(ongaro): There are probably two things missing from this header. First,
00048  * it should have some mechanism (such as a version field) for changing the
00049  * header format later. Second, it should have a checksum, as the
00050  * TCP/IP/Ethernet checksums may not be sufficient.
00051  */
