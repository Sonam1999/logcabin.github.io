00025 /**
00026  * A File is called by the Event::Loop when a file becomes readable or
00027  * writable. The client should inherit from this and implement the
00028  * handleFileEvent() method for when the file becomes readable or writable.
00029  *
00030  * File objects can be created from any thread, but they will always fire on
00031  * the thread running the Event::Loop.
00032  */

00037      * Construct and enable monitoring a file descriptor.

00039      *      Event::Loop that will manage this File object.

00042      *      is called first, this File object will close the file descriptor
00043      *      when it is destroyed.

00045      *      Invoke the handler when any of the events specified by this
00046      *      parameter occur (OR-ed combination of EPOLL_EVENTS values). If this
00047      *      is 0 then the file handler starts off inactive; it will not trigger
00048      *      until setEvents() has been called (except possibly for errors such
00049      *      as EPOLLHUP; these events are always active).
